라이브러리 의존성 관리

- 기존에 외부 방식으로 .js 파일을 가져와서 쓰고자 할 때, head 태그 영역에 script 태그를 이용해서 연동해서 사용했었음!!
예) jQuery
- 외부 방식으로 .js 파일을 가져오고자 할 때, 자바처럼 "import" 구문을 이용해서 .js 를 연동해서 사용할 수 있는 문법
- CDN 방식을 대체한다. (온라인 방식)


* 기존의 CDN 방식 (script 태그 이용)

- head 영역에 script 태그를 이용해서 src 속성에 가져다 쓰고자 하는 .js 파일의 url 주소를 기술


* 라이브러리 의존성 관리 방식 (import 구문 이용)

- 기존의 script 태그를 이용한 CDN 방식을 대체하여 자바처럼 "import" 구문을 이용해서 .js 파일의 url 경로를 기술


* 내가 직접 만든 .js 모듈을 import 해보기

- import 구문은 CDN 방식만 대체한다!! 오프라인 방식은 불가!!
- 보안상의 이유로 내가 직접 갖고있는 .js 파일을 연동할 경우 접근이 막혀서 제대로 동작이 되질 않는다!! (file://)
해결방법 : 이 컴퓨터를 서버로 만들자!! (http://~~~.js)

* VSCode 상에서 간단히 웹서버를 구축할 수 있다!!
- VSCode 마켓플레이스에서 "live server" 확장팩을 설치한다.



* 클래스 단위도 export, import 할 수 있음!!
